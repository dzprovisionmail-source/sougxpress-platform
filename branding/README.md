# SougXpress Platform — Brand Identity Foundation

## Overview
This directory is the official, permanent visual source of truth for the SougXpress Platform. It serves as the central repository for all brand assets, identity guidelines, and production resources used across mobile, web, and physical environments.

## Purpose of Branding
Branding is the emotional and visual DNA of SougXpress. While code defines how the platform works, Branding defines who we are, what we promise, and how we are perceived by the citizens of Ain Sefra and beyond.

### Key Distinctions
- **Branding vs. Design System**: Branding defines the soul, story, and core visual assets (Logo, Mascot, Brand Colors). The Design System (in `docs/design-system/`) translates these into UI components, spacing rules, and code tokens for developers.
- **Branding vs. UI**: Branding is the identity; UI is the interface. UI uses brand assets to create functional screens.
- **Branding vs. Marketing**: Branding is the permanent foundation. Marketing is the temporary application of this brand to specific campaigns, social media posts, and advertisements.

## Directory Structure
- `origin/`: Historical inspirations and the evolution of the brand.
- `brand-book/`: The narrative and philosophical foundation of the identity.
- `logo/`: Official vector and raster versions of the SougXpress logo.
- `mascot/`: Official character designs and expressions.
- `colors/`: The official semantic palette and design tokens.
- `typography/`: Arabic and Latin type scales and font files.
- `ui-assets/`: Specialized assets for app splash screens, onboarding, and loading states.
- `marketing/`: Resources for social media and advertising campaigns.
- `delivery/`: Physical branding for uniforms, vehicles, and packaging.
- `archive/`: Preservation of previous versions to ensure traceability.

## Governance
1. **Source of Truth**: No visual asset is official until it is committed to this directory.
2. **Permanence**: Core brand files (Logos, Brand Values) should remain stable. Changes require Founder approval.
3. **Independence**: This directory is separate from documentation (`docs/`) as these are production-ready resources.
